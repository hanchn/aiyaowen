# 教育类短视频 APP 改造计划

既然这是一个学习 APP，我们需要将现有的“泛娱乐”风格调整为“泛知识/教育”风格。核心在于内容展示的专业性和学科分类。

## 1. 数据模型与 Mock 数据更新 (`features/home/data`)
- **学科分类**：引入 `subject` 字段 (物理、化学、生物、英语、数学)。
- **内容调整**：将 Mock 数据替换为教育相关内容（例如：物理实验视频、数学公式推导图片、英语单词测验表单）。
- **Tag 系统**：增加知识点标签（如 `#力学`, `#三角函数`, `#细胞分裂`）。

## 2. 首页 Feed 流改造 (`features/home`)
- **顶部导航栏**：增加“学科”筛选 Tab (全科/物理/数学/...)，点击可筛选 Feed 流。
- **UI 调整**：
    - 底部信息栏：增加“知识点/难度等级”显示。
    - 互动栏：将“点赞”改为“收藏/标记掌握”。

## 3. 探索页改造 (`features/explore`)
- **知识图谱/分类索引**：将探索页改为学科分类入口。
- **搜索功能**：强化搜索，支持按知识点搜索。

## 4. 个人中心改造 (`features/profile`)
- **学习档案**：展示“学习时长”、“已掌握知识点”。
- **错题本/收藏夹**：Tab 改为“我的收藏”和“错题本”。

## 执行步骤
1.  **更新数据结构**：修改 `FeedItem` 接口，增加学科属性，并更新 `MOCK_FEED` 数据。
2.  **改造首页 UI**：添加顶部学科筛选器，优化视频/图片展示的信息层。
3.  **改造探索页**：实现基于学科的分类网格导航。
4.  **调整个人中心**：将娱乐化的统计数据改为学习相关的统计。
